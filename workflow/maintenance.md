# maintenance.md — 知识库维护清单（定期扫库）

> 触发时机：不是写新页时，而是**隔一段时间**或**库感觉脏了**时。  
> 目标：防止「没人入库时，知识库照样静默腐烂」。  
> 与 `intake.md` 的分工：入库是事件驱动（跟着新页）；维护是时间/规模驱动（扫整库）。

## 总闸

- 本文件是**清单**，不是第二套入库流程。不新建知识页、不改 kind 定义。
- 改状态、改索引、改术语表时，规则仍以 `schema.md` 为准。
- 扫完后建议在 `analysis/log.md` **追加一行**维护记录（可选但推荐），格式示例：

  ```text
  [2026-07-19] maintenance | staleness+orphan+tags 扫完；标 stale: 0；死链: 0；glossary 补: 1 | 无
  ```

## 何时跑

满足**任一**即可开跑（不必等全部）：

- 自上次维护后，大约又入库了 **5–10** 个新页；或
- 距上次维护约 **1 个月**；或
- 工具链大版本 bump / 核心研究对象行为明显变了；或
- 维护者明确说「扫一下库」。

库还很小（个位数页面）时：可只做第 1、2 项，第 3 项快速过一眼即可。

## 第 1 项 · Staleness（事实 / 版本是否还立得住）

**问题**：`status: active` 的页挂着旧 `applies_to` 或过时事实，检索时仍当「当前真理」。

**做什么**：

1. 列出所有 `status: active` 的知识页（可从 `analysis/index.md` 出发，再对一下 frontmatter）。
2. 对每页看：`applies_to` 是否仍覆盖当前在用的工具链/版本；正文关键断言是否已被新页或外部事实否定。
3. 裁决（人工或 agent 提议、维护者拍板）：
   - 仍成立 → 不动（可选：仅刷新 frontmatter `updated` 若你顺手复核了全文）。
   - 部分过时但结论骨架还在 → 就地修订，或标 `stale` 并开新页。
   - 已被替代 → `superseded`，`note:` 指向新页（见 `schema.md` 第 4 节）。

**不做**：

- 不把「很久没改」自动当成 stale（久 ≠ 错）。
- 不在本步 bulk 重写全文；先标状态或列待办。

## 第 2 项 · Orphan / 死链（索引与文件是否一致）

**问题**：`index` 与磁盘脱节 → 检索进死胡同，或页写了等于没入库。

**做什么**：

1. **死链**：`analysis/index.md` 里每一行链接的文件是否还存在。
2. **孤儿页**：`analysis/` 下应被索引的知识页（`concepts/` / `mechanisms/` / `cases/` / `decisions/` 及约定纳入的目录；**不含**纯原料草稿）是否都在 `index.md` 里有行。
3. 处理：补索引 / 删死链行 / 确认删除的页是否应在 `log.md` 留痕。

**`PriorKnowledge/`**：历史遗留，**默认纳入**死链与孤儿检查（已在 index 的要能打开；若决定永不索引，在本条维护记录里写明例外，避免反复吵）。

## 第 3 项 · Tag ↔ Glossary 一致性

**问题**：同义词漂移、术语表空转、入库 Step 6 管不到历史页。

**做什么**：

1. 抽一遍知识页 frontmatter 的 `tags`：是否都在 `analysis/glossary.md` 有定义。
2. 反查：glossary 里的词是否至少被一页用到（长期零引用可标「待删 / 待合并」，不必当场删）。
3. 同义词：同一概念两种 tag → 统一到 glossary 主词，旧 tag 在 glossary 写「见 xxx」或改页。

**不做**：不为了漂亮批量发明新 tag 体系。

## 明确不做（本清单范围外）

| 不做 | 原因 |
|------|------|
| 强制「相关页面」双向回链 | 小库成本高；单向相关足够时别为完备性加班 |
| 自动 relation mining / RAG / 向量库 | 违反本库「纯 Markdown、不引入 RAG」取向 |
| 重跑完整 intake 六步去「修复」旧页 | 维护是体检与状态修正，不是伪入库 |
| 回改 `log.md` 历史行 | 日志只追加（`schema` / intake 纪律） |

## 跑完检查（最短）

- [ ] 第 1 项：active 页过了一遍，该标的 `stale`/`superseded` 已标或已记「暂不处理」
- [ ] 第 2 项：index 无死链；应索引页无孤儿（或已记录例外）
- [ ] 第 3 项：本轮用到的 tag 与 glossary 对齐
- [ ] （推荐）`log.md` 追加一条 maintenance 记录

**任一项只做了一半就中断 → 在 log 或当面说明扫到哪，下次从断点续，别假装整库健康。**
